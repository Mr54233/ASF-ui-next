## 🔄 工作流程

### 1. 开始新功能

```
1. 查看 [需求跟踪表] 确认需求编号和优先级
2. 查看 [实施文档] 了解详细实施步骤
3. 参考老版本代码: D:\_workstation\ASF-ui\
4. 开始实施
5. 更新 [需求跟踪表] 状态
6. 更新 [实施文档] 笔记
```

### 2. 更新进度

当需求状态发生变化时，需要更新以下文档：

1. **[需求跟踪表]** - 更新对应需求的 `状态`、`实际工时`、`完成日期`
2. **[实施文档]** - 添加实施笔记、更新进度
3. **[功能完成度清单]** - (可选) 更新整体完成度

### 3. 创建新的实施文档

```bash
# 在 docs/实施/ 目录下创建新文档
命名格式: {模块名}实施文档.md

参考模板: docs/实施/_模板.md
```
