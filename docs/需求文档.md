## 一、项目概述

### 1.1 项目目标

ASF-ui-next 是 ArchiSteamFarm 的 Web UI 界面重构项目，目标是用 Vue 3 + Element Plus 重写原有的 Vue 2 版本，提供现代化、美观、高性能的 ASF 管理界面。

### 1.2 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| Vue | 3.x | Composition API |
| TypeScript | 5.x | 严格类型检查 |
| Element Plus | 最新 | UI 组件库 |
| Pinia | 最新 | 状态管理 |
| Vue Router | 4.x | 路由管理 |
| Vite | 5.x | 构建工具 |
| ECharts | 5.x | 图表库 |
| UnoCSS | 最新 | 原子化 CSS |

### 1.3 设计规范

- **玻璃态设计** (Glassmorphism) - 主视觉风格
- **响应式设计** - 支持桌面/平板/移动端
- **深色模式** - 支持 light/dark/auto 三种模式
- **动画过渡** - 流畅的页面和交互动画

## 二、核心功能需求

### 2.1 认证系统 (Priority: P0)

#### 2.1.1 登录流程

```typescript
// API: POST /Api/Auth
interface LoginRequest {
  Password: string  // ASF IPC 密码
}

interface LoginResponse {
  Success: boolean
  Message?: string
  Token?: string  // JWT token 或会话标识
}
```

**需求:**
1. 用户首次访问显示登录页面
2. 输入 IPC 密码后调用登录 API
3. 登录成功后保存 token 到 localStorage
4. 跳转到默认页面 (Dashboard 或 Bots)
5. 登录失败显示错误提示

#### 2.1.2 认证守卫

```typescript
// 路由守卫逻辑
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('asf-token')
  const requiresAuth = to.meta?.requiresAuth !== false

  if (requiresAuth && !token) {
    next({ name: 'login', query: { redirect: to.fullPath } })
  } else if (to.name === 'login' && token) {
    next({ name: 'default' })
  } else {
    next()
  }
})
```

#### 2.1.3 自动登录

- token 有效期内自动登录
- token 过期自动跳转登录页

```
GET    /Api/Bot/{bots}/BGR             # 获取 BGR 状态
POST   /Api/Bot/{bots}/BGR/Add         # 添加 Key
POST   /Api/Bot/{bots}/BGR/Delete      # 删除 Key
POST   /Api/Bot/{bots}/BGR/Clear        # 清空队列
POST   /Api/Bot/{bots}/BGR/Reset        # 重置状态

POST   /Api/Command                    # 发送命令
```
